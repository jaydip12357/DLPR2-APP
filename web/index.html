Common questions

Is my child's data private?
Messages are stored in a secure database and used only for threat analysis. The system is designed for a single family; we do not sell or share data.
What about false positives?
Analysis is tuned to reduce false positives. Parents always see the full message and flag reason so you can judge for yourself.
Does it work on iPhone?
No. SafeType requires Android (keyboard and accessibility features). iOS does not allow equivalent monitoring without enterprise MDM.
How do I get the app?
Follow the Setup Guide to download the APK from our secure link and install it on your child's Android device.

Model & research

How SafeType detects harmful content and what we use today. Part of our Duke University Deep Learning project.

Our custom model (production)

SafeType uses our custom content-safety model developed for the Duke Deep Learning project. It is deployed and live at:

SafeType AI Keyboard Safety Monitor — Live API

This app (parent dashboard and Android capture) is linked to that API. Messages are sent to the custom model for classification; results (label and confidence) are used to flag threats on the dashboard.

  • Live API: https://dl-project-2-second-version.onrender.com
  • Source repo: DL_Project_2-Second_version- (code, training, data)
  • Input: Message text (and optional batch).
  • Output: Predicted label (one of 10 classes), confidence, and top-3 class probabilities.

Threat categories detected (10 classes)

Our custom model classifies messages into these 10 categories:

  • clean — Safe, non-harmful content
  • racism — Racial hate or discrimination
  • sexism — Sexist or misogynistic content
  • profanity — Obscene or swear words
  • cyberbullying — Insults, personal attacks
  • toxicity — Generally toxic language
  • hate_speech — Generic hate speech
  • implicit_hate — Implicit or indirect hate
  • threat — Threats, aggression
  • sarcasm — Sarcastic or ironic (context-dependent)

Duke Deep Learning project — custom model repo

The custom SafeType content-safety models are developed in a separate repository: DL_Project_2-Second_version-. It implements an NLP system that detects 10 categories of harmful or sensitive content for children in online chat.

Models (10-way classification)

Type Model Artifacts Accuracy Weighted F1
Naive baseline Keyword-matching + majority class naive_baseline.pkl ~0.21 ~0.17
Classical ML TF-IDF + Logistic Regression logistic_regression.pkl, tfidf_vectorizer.pkl ~0.63 ~0.62
Neural Fine-tuned DistilBERT distilbert-toxicity/ ~0.72 ~0.72

Label taxonomy (10 classes)

clean, racism, sexism, profanity, cyberbullying, toxicity, hate_speech, implicit_hate, threat, sarcasm.

Training data (8 sources, balanced to 50k)

  • Davidson et al. — 24,783 tweets (labeled_data.csv)
  • Jigsaw Toxic Comment v1 — 159,571 comments (multi-label)
  • Jigsaw Unintended Bias v2 — 100,000 sampled
  • UC Berkeley D-Lab Measuring Hate Speech — 39,565 comments (HuggingFace)
  • EDOS — Explainable Detection of Online Sexism, 20,000 posts (Gab & Reddit)
  • HateXplain — 20,148 posts with target-community annotations
  • ImplicitHate — 6,346 implicit hate tweets (HuggingFace)
  • TweetEval irony — ~4,601 tweets for sarcasm/irony (HuggingFace)

Combined ~375k samples → balanced to 50,000 (5,000 per class). Pipeline: make_dataset.py → build_features.py (TF-IDF, handcrafted features) → model.py (train all three) → experiment.py (training-size sensitivity, noise robustness). Inference: main.py (interactive / single / batch); app.py for API.

Setup Guide

Get SafeType running on your child's device in under 10 minutes. You'll need physical access to their Android phone.

1

Download the APK

Download the SafeType installer from our secure link. The file is approximately 15 MB.

Download from Google Drive
2

Enable installation from unknown sources

On the child's device, go to Settings → Security → Install unknown apps and allow your file manager or browser to install APKs. This can be disabled again after installation.

3

Install and grant permissions

Open the downloaded APK and install SafeType. The app will request three permissions:

  • Accessibility Service — reads on-screen messages from chat apps
  • Notification Access — captures incoming message notifications
  • Input Method — enables the SafeType keyboard to capture outgoing text

All three must be enabled for full monitoring coverage.

4

Set SafeType as the default keyboard

Go to Settings → System → Languages & input → Default keyboard and select SafeType. This enables outgoing message capture.

5

Verify on your dashboard

Send a test message from the child's device. Within 10–15 seconds, it should appear on your parent dashboard. The device status will show as "Online".

Transparency: SafeType shows a persistent notification on the child's device that says "SafeType Parental Monitor is active." This is intentional — children should know they are being monitored.
